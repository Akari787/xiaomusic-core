v1.1.0 发布说明 ​

发布日期：2026-04-17
适用分支：main

1. 本次发布的基线 ​

v1.1.0 的描述基于 v1.0.10 tag 与当前工作树的代码差异，不是沿用口头结论倒推。

本次对比基线：

  • 旧版本：v1.0.10
  • 当前版本：v1.1.0
  • 参考命令：git diff --stat v1.0.10..HEAD

当前差异统计显示：

  • 39 个文件发生过代码/文档层变更
  • 主要落点集中在：auth.py、diagnostics.py、api/routers/system.py、api/routers/v1.py、playback/facade.py、device_player.py、source_plugin_manager.py、WebUI 新页面与相关测试

因此，v1.1.0 不是 v1.0.10 那种以文档收口为主的版本；它相对 v1.0.10 已经新增了正式接口、诊断视图、来源插件管理能力，以及一批围绕 Jellyfin / playlist / player state 的真实实现修正。

2. 相对 v1.0.10 的真实变化 ​

2.1 Auth：从“主线文档收口”推进到“正式状态接口 + 短会话重建实现” ​

相对 v1.0.10，当前代码已新增并收口以下能力：

  • 新增正式端点：GET /api/v1/auth/status
  • system.py 与 v1 路由不再各自拼接公开状态，而是复用统一状态映射来源
  • auth.py 新增：
    • auth_public_status_snapshot(...)
    • map_auth_public_status(...)
    • rebuild_short_session_from_persistent_auth(...)
    • MiAccount 主路径重建
    • MiJia fallback 重建
    • 阶段化 debug flow 记录
  • short session 缺失场景不再只停留在“需要登录”的描述层，而是接入实际 rebuild 主路径

这代表：

  • v1.0.10 主要在说明当前 auth 主线与验收边界
  • v1.1.0 已把其中一部分关键恢复链落成可调用、可观测、可测试的正式实现

当前仓库同时新增了对应说明文档：

  • docs/architecture/auth_runtime_recovery.md

2.2 Diagnostics：/diagnostics 从启动信息碎片升级为统一诊断视图 ​

相对 v1.0.10，GET /diagnostics 不再只是 startup/self-check 的零散信息出口，而是统一聚合为一份启动/可用性摘要视图。

当前代码中的统一视图包含：

  • startup
  • auth
  • sources
  • devices
  • playback_readiness

并统一输出：

  • generated_at_ms
  • overall_status
  • summary
  • areas

状态枚举固定为：

  • ok
  • degraded
  • failed
  • unknown

同时，WebUI 已新增最小诊断页：

  • xiaomusic/webui/src/pages/DiagnosticsPage.tsx
  • xiaomusic/webui/src/services/diagnostics.ts

这说明 v1.1.0 相对 v1.0.10 已从“文档化当前状态”推进到“正式诊断视图 + 最小 UI 消费”。

2.3 Sources：来源插件进入可观察、可操作阶段 ​

相对 v1.0.10，当前代码已不再只有静态来源注册，而是新增了最小来源插件管理层。

当前新增内容包括：

  • SourcePluginManager
  • active registry snapshot
  • 来源观察类 API：
    • GET /api/v1/sources
    • POST /api/v1/sources/reload
  • 来源可操作 API：
    • POST /api/v1/sources/upload
    • DELETE /api/v1/sources/{name}
    • POST /api/v1/sources/{name}/enable
    • POST /api/v1/sources/{name}/disable
  • WebUI 最小 sources 管理页：
    • xiaomusic/webui/src/pages/SourcesPage.tsx
    • xiaomusic/webui/src/services/sources.ts

当前边界也需要如实说明：

  • disabled_plugins 仍是进程内内存态，不保证重启后保留
  • 当前版本实现的是最小热加载/管理骨架，不是完整插件平台

2.4 Playback / Player State：Jellyfin 与 playlist 主链路相对 v1.0.10 已有实质推进 ​

相对 v1.0.10，当前播放链与状态面发生了实质变化，重点不在“新增更多播放入口”，而在于统一状态源、补齐 playlist-aware external URL 主链、减少 WebUI 与设备真实态之间的偏差。

当前代码中已落地的关键点包括：

  • build_player_state_snapshot() 成为 /api/v1/player/state 与 /api/v1/player/stream 的统一状态来源
  • track.source 新增规范化与 session-aware fallback
  • Jellyfin URL 归属判断下沉到共享 classifier + plugin 能力声明，不再走 registry 站点特判
  • LocalLibrarySourcePlugin 可在 playlist context 下解析 Jellyfin-synced playlist item
  • external URL 播放链已支持 playlist-aware session bootstrap，同步：
    • cur_playlist
    • _play_list
    • _current_index
    • cur_music
  • external URL 成功后补齐 post-start hook：
    • duration
    • next timer
    • volume
    • PLAYER_STATE_CHANGED
  • /api/v1/player/state 当前返回中已补入 volume
  • WebUI 首页已针对以下问题做过最小收口：
    • playlist 选中态
    • 进度条基线刷新
    • 切歌后进度重置
    • Jellyfin playlist 场景的 state 消费一致性

这意味着，v1.1.0 相对 v1.0.10 已经不是“只确认首轮 start confirmation 的局部证据”，而是继续把 playback 主链中的若干真实断点推进到了代码与实机都可验证的状态。

4. 当前已确认的范围 ​

以下内容在当前版本中已有代码事实，且本轮工作中已分别完成过定向测试、构建验证或测试服务器实机验证：

4.1 Auth ​

  • /api/v1/auth/status 已存在并作为正式 v1 端点暴露
  • system / v1 / debug 已共享统一公开状态映射来源
  • short-session 缺失后的 persistent-auth rebuild 主路径与 fallback 路径已接入

4.2 Diagnostics ​

  • GET /diagnostics 已返回统一五维诊断视图
  • WebUI 已存在最小诊断页入口与消费逻辑

4.3 Sources ​

  • source plugin manager 已存在
  • /api/v1/sources 观察/管理接口已存在
  • WebUI 已存在最小 sources 管理页

4.4 Playback ​

  • /api/v1/player/state 与 /api/v1/player/stream 已共享 snapshot 来源
  • Jellyfin / local_library playlist-aware 主链已可维护 context 与 current_index
  • external URL 播放成功后已补齐 duration / timer / volume 运行态建立
  • /api/v1/play 与 /api/v1/control/next 相关的重复 player_state 双保险推送已完成收口

5. 当前未完成或仍需如实保留的边界 ​

v1.1.0 不应被写成“所有相关问题都已彻底完成”。当前仍需保留以下边界：

  • disabled_plugins 仍非持久化
  • M2 P2（legacy/source capability 进一步压缩与能力声明驱动迁移）未完成
  • 浏览器自动化仍有偶发脚本失败，不能把其稳定性写成已完全解决
  • 旧的部分 HomePage vitest 用例仍存在测试夹具/挂载漂移，不应写成“前端测试全绿”
  • 当前实机验收覆盖的是关键路径，不等于所有来源、所有模式、所有网络条件下的长期稳定性总验收

6. 本次发布确认的是哪一层，不是哪一层 ​

本次发布确认的是 ​

  • auth 恢复链从文档主线推进到正式状态接口与实际 rebuild 主路径这一层
  • startup/self-check 诊断从零散返回推进到统一诊断视图与最小 WebUI 消费这一层
  • 来源插件从静态注册推进到最小 manager + API + WebUI 管理这一层
  • Jellyfin / playlist / player state 若干关键断点已完成主链修补与实机闭环这一层

本次发布没有确认的不是 ​

  • 来源插件平台的最终形态
  • 所有播放路径与所有站点来源的长期稳定性总验收
  • 所有浏览器自动化与前端旧测试夹具问题都已解决
  • auth / playback / source plugin 全边界一次性彻底收口
