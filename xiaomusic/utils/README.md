# 目录职责

本目录提供跨业务领域复用的通用工具函数，按功能领域分文件组织。工具函数应是无副作用或副作用明确的独立函数，不持有业务运行时状态。

# 文件说明

## file_utils.py

文件与目录操作工具。

主要内容：
- 目录深度路径计算
- 文件扫描与过滤（音乐文件扩展名检测）
- 目录创建、文件复制、路径拼接等通用操作

典型使用方：`music_library.py`（媒体库扫描）

## network_utils.py

网络请求与下载工具。

主要内容：
- 基于 aiohttp 的异步 HTTP 下载（经 `OutboundPolicy` 安全策略）
- 带缓存的 URL 内容获取
- Edge TTS 语音合成调用封装
- URL 哈希与文件名生成

典型使用方：`device_player.py`（TTS 下载）、`online_music_service.py`

## music_utils.py

音乐文件元数据处理工具。

主要内容：
- 音频文件格式检测（MIME 类型推断）
- 音乐文件元数据读取（标题、艺术家、专辑封面）
- Base64 封面图像提取
- 播放列表文件解析（M3U 等格式）
- 媒体文件哈希计算

典型使用方：`music_library.py`（媒体库索引）

## system_utils.py

系统操作与运行环境工具。

主要内容：
- 异步子进程执行封装
- 平台环境检测（Docker / 架构检测）
- 自更新下载与解压（受 `SelfUpdateDisabledError` 保护）
- Cookie 与 HTTP 会话工具
- 随机字符串生成

典型使用方：`xiaomusic.py`（版本更新）、`relay/`（进程调用）

## text_utils.py

文本处理与搜索工具。

主要内容：
- 繁简中文转换（基于 `OpenCC`）
- 字符串相似度计算（`difflib`）
- TTS 文本预处理（标点清理、分句）
- 异步文本流处理

典型使用方：`device_player.py`（TTS 文本准备）、`conversation.py`

## openai_utils.py

AI 大模型调用工具（可选功能）。

主要内容：
- OpenAI 兼容接口异步调用封装
- 音乐播放指令解析提示词（从自然语言中提取歌名 / 歌手）
- 超时与错误处理

典型使用方：`conversation.py`（自然语言命令解析，可选路径）

# 不应该放什么

- 业务逻辑（播放控制、设备管理、认证流程等）
- 持有运行时状态的对象（类实例、全局缓存除外明确的工具缓存）
- HTTP 路由或 API 模型
- 安全策略（属于 `security/`）

# 使用原则

- 工具函数应可独立测试，不依赖运行时全局状态。
- 若某个工具函数开始依赖 `Config`、`DeviceManager` 等运行时对象，应考虑是否归类错误。
- 新增工具时，按领域选择合适的现有文件；若领域不明显，优先放入最接近的现有文件，而非新建文件。
