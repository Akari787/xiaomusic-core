## 2. 相对 v1.1.0 的真实变化

### 2.1 WebUI 歌单选择状态彻底整改

v1.1.0 的 WebUI pending selection 生命周期逻辑在实机验收时暴露了结构性缺陷：歌曲 selector 使用 `key={item.id}`，但后端 `/api/v1/library/playlists` 返回的 track id 可能重复，导致 React `<option>` 残留、列表重复/错位/不可选。

本次整改不止于前端止血，而是彻底引入 identity 体系：

- 前端：`HomePage.tsx` 歌曲 selector key 改为 `${effectivePlaylist}:${item.id}:${idx}`
- 后端：建立 `music_entities` / `playlist_memberships` / `playlist_definitions` 主模型，替代 `all_music={name:url}` / `music_list={playlist:[name]}` 的 name-based 旧结构
- API：`/api/v1/library/playlists` 优先输出 membership 结构与 `entity_id`，不再基于 filepath/url 生成重复 track id

### 2.2 设备运行态 identity 化

`config.py` 的 `Device` 类新增三个显式 identity 槽位：

- `current_entity_id`
- `current_playlist_item_id`
- `current_display_name`

`device_player.py` 开始维护结构化 `_play_list_items`（list[dict]），并提供：

- `_build_playlist_runtime_items()`：从 membership 或 legacy `music_list` 构建结构化列表
- `_find_playlist_index()`：**优先按 `playlist_item_id → entity_id → display_name → legacy_name → index fallback`** 定位，不再仅靠 `_play_list.index(name)`
- `_set_runtime_track_reference()` / `get_current_track_reference()`

external URL bootstrap 也已改为优先按 `playlist_item_id → entity_id → display_name` 对位。

### 2.3 Legacy API identity 收口

`/playingmusic` 与 `/ws/playingmusic` 改为基于 `PlaybackFacade.build_player_state_snapshot()` 投影，兼容返回中新增：

- `entity_id`
- `playlist_item_id`
- `current_index`
- `context_id`

`/api/v1/library/music-info` 与 `/musicinfo` 开始接受 `entity_id` 查询；若只给 `entity_id`，通过 `get_legacy_name_for_entity()` 解析展示名再回查 URL/tags。

### 2.4 Favorites entity 全链路

- `FavoritesRequest` 新增 `entity_id` 字段，`/api/v1/library/favorites/add|remove` 优先透传 `entity_id`，避免同名歌曲加入/移出收藏时继续只靠 `track_name` 判等
- 旧 `/playlistaddmusic` 等路由的 `PlayListMusicObj.music_list` 开始接受 `{entity_id, display_name}` 结构，便于老入口平滑过渡到 entity 引用
- `play_list_add_music('收藏', ...)` 修复：`收藏` 原被保留名拦截无法自动创建；新建歌单后现在会重新获取 `custom_play_list` 引用，避免后续 append 写到旧引用里导致条目丢失

### 2.5 Source plugin identity 接受能力

`PlaybackFacade` 在标题兜底前优先解析 `entity_id`，`local_library_source_plugin` 优先接受 payload `entity_id`。

## 5. 当前未完成或仍需如实保留的边界
